# VLM 视频审计能力深度调研报告

**调研日期**: 2026-03-01  
**目的**: 评估 VLM 是否能够对实训视频进行动作检测与打分

---

## 📊 核心结论

### 能做吗？

**✅ 可以做，但有明确的能力边界和限制。**

| 能力维度 | 当前水平 | 我们的需求 | 匹配度 |
|----------|----------|------------|--------|
| 物体识别 | 强 | 识别灭火器、工具等 | ✅ 完全匹配 |
| 基础动作识别 | 中等 | 识别拔、握、压等动作 | ✅ 基本匹配 |
| 细粒度动作判断 | 弱 | 判断握持位置是否正确 | ⚠️ 需要优化 |
| 时序推理（步骤顺序） | 中等 | 判断操作顺序是否正确 | ⚠️ 需要设计 |
| 动作质量评估（AQA） | 研究前沿 | 给每个步骤打分 | ⚠️ 需要定制 |

---

## 🔬 关键发现

### 1. 当前 VLM 的视频理解能力

#### Gemini 2.5 Pro（2025年5月）
- **VideoMME 准确率**: 85.2%（视频理解基准测试 SOTA）
- **时刻检索 (QVHighlights)**: R1@0.5 接近 SOTA
- **能力亮点**:
  - 可处理最长 6 小时视频（200万 token）
  - 能识别视频中的 16+ 个独立片段
  - 能数清视频中特定动作的出现次数（如"使用手机17次"）
  
**来源**: Google Developers Blog, "Advancing the frontier of video understanding with Gemini 2.5" (2025-05)

#### GPT-4o / GPT-4V
- 在大多数视频理解任务上表现强劲
- **但**：在细粒度时序推理上与 Gemini 2.5 有差距
- 视频处理限制：API 最长支持约 3 分钟视频

#### 关键限制（所有 VLM 共有）
> **"fast action sequences might lose detail due to the 1 FPS sampling rate"**  
> — Google Gemini API 文档

- **采样率限制**：默认 1 FPS，快速动作会丢失细节
- **细粒度感知差距**：VLM 在精确定位上仍有明显短板
- **时序边界模糊**：在高层池化/token 合并时，局部细节和事件边界可能丢失

---

### 2. TemporalBench 基准测试揭示的真相

**TemporalBench**（2024年10月发布）是专门评估细粒度时序理解的基准：

| 模型 | QA 准确率 |
|------|-----------|
| **人类** | 67.9% |
| **GPT-4o** | 38.5% |
| **其他 SOTA 模型** | 更低 |

**关键发现**：
- 人类与 AI 在时序理解上存在 **~30% 的差距**
- 模型能理解静态视觉概念，但在细粒度时序关系推理上仍有限制
- 细粒度时序差异（如"切了三次"vs"切了两次"）是主要挑战

---

### 3. Action Quality Assessment (AQA) 研究现状

**AQA 是一个活跃的研究领域**，过去十年有 200+ 篇论文发表（IJCV 2025 综述）。

#### 应用场景
- **体育**: 跳水、体操、花滑评分
- **医疗**: 手术技能评估、康复训练
- **职业培训**: 工业操作、安全规范

#### 技术路线
1. **基于规则的评分**: 将动作分解为步骤，检查每步完成情况
2. **基于回归的评分**: 直接预测连续分数
3. **基于排名的评分**: 比较两个表现的优劣

#### 当前 SOTA 方法
- **不是用 VLM**，而是专门的视频理解模型 + 监督学习
- 需要大量标注数据（正确/错误示例 + 分数）
- 在特定领域（如跳水）能达到与人类评分 r ≥ 0.80 的相关性

---

### 4. 护理/医疗领域的 VLM 应用（最相关案例）

**论文**: "Automated Procedural Analysis via Video-Language Models for AI-assisted Nursing Skills Assessment" (arXiv 2509.16810, 2025-09)

**关键发现**:
- VLM 可以分析复杂的顺序性活动
- 能理解时序关系
- 能生成上下文感知的反馈
- **优于传统动作识别模型**：因为 VLM 能理解文字描述的标准

**但**：这是一个研究项目，不是开箱即用的产品。

## 🎯 针对职校实训的可行性评估

### 场景分析：消防灭火器操作

| 评估点 | VLM 能力 | 实现难度 | 备注 |
|--------|----------|----------|------|
| 检测灭火器 | ✅ 强 | 低 | 通用物体识别 |
| 检测人物动作（站立、弯腰） | ✅ 强 | 低 | 姿态识别成熟 |
| 检测是否拔出保险销 | ⚠️ 中等 | 中 | 小物体，需要高分辨率 |
| 判断握持位置是否正确 | ⚠️ 弱 | 高 | 细粒度空间判断 |
| 判断喷射方向是否对准火焰根部 | ⚠️ 中等 | 中 | 空间关系推理 |
| 判断步骤顺序是否正确 | ⚠️ 中等 | 中 | 时序推理 |
| 综合打分 | ⚠️ 研究前沿 | 高 | 需要定制 |

### 技术方案建议

#### 方案 A: 纯 VLM 方案（推荐初期采用）

```
视频 → 抽帧 (1-2 FPS) → VLM 逐帧/片段分析 → 规则整合 → 评分
```

**优点**:
- 开发快，成本低
- 利用现有 VLM API（Gemini 2.5 Pro）
- 适合 MVP 验证

**限制**:
- 细粒度判断准确率有限
- 依赖 Prompt 工程质量
- 需要高质量的评分标准文档

**预期准确率**: 60-75%（与人工评分对比）

#### 方案 B: VLM + 专用模型混合方案

```
视频 → 专用模型（骨骼检测、物体追踪） → VLM 语义理解 → 融合评分
```

**优点**:
- 细粒度判断更准确
- 可解释性更强

**限制**:
- 开发周期长
- 需要更多领域数据训练专用模型

**预期准确率**: 75-85%

#### 方案 C: 完全定制的 AQA 模型

```
视频 → 视频编码器 → AQA 回归模型 → 评分
```

**优点**:
- 理论上准确率最高（r ≥ 0.80 目标可达）

**限制**:
- 需要大量标注数据（500+ 视频，每个步骤的分数）
- 开发周期 6+ 个月
- 成本远超 20 万预算

## 📋 20万预算下的推荐方案

### 采用方案 A（纯 VLM），分阶段实施

**Phase 1: MVP 验证（2个月）**
1. 选择 1-2 个实训场景（如消防灭火器）
2. 使用 Gemini 2.5 Pro API
3. 设计 Prompt + 评分规则
4. 目标：验证技术可行性，准确率 > 60%

**Phase 2: 优化迭代（2个月）**
1. 根据错误案例优化 Prompt
2. 引入 few-shot 示例（正确/错误视频截图）
3. 添加人工复核机制（低置信度案例）
4. 目标：准确率提升到 70-75%

### 关键成功因素

1. **高质量评分标准**
   - 老师必须提供详细的、可量化的评分标准
   - 每个步骤的正确/错误判定标准要明确

2. **示范视频**
   - 正确操作的标准视频
   - 常见错误的示例视频
   - 用于 Prompt 中的 few-shot 示例

3. **视频质量要求**
   - 固定机位，画面稳定
   - 分辨率 ≥ 720p
   - 关键动作区域要清晰可见

4. **人工复核机制**
   - AI 评分 + 置信度
   - 低置信度（< 0.7）的案例需人工复核
   - 这是 20万预算内保证质量的关键

## 🚨 风险与应对

| 风险 | 概率 | 影响 | 应对措施 |
|------|------|------|----------|

## 🎯 结论

**VLM 能做实训视频审计，但不是"开箱即用"。**

- ✅ 技术上可行，有相关研究和案例支撑
- ⚠️ 细粒度判断（如握持位置）是挑战，需要额外工程
- ⚠️ 20万预算内，建议采用 VLM + 人工复核的混合模式
- ❌ 不要承诺"完全自动化"或"100%准确"

**建议在项目 Kick-off 时与老师明确：**
1. AI 评分是辅助工具，不是替代人工
2. 初期准确率目标 70-75%，后续迭代提升
3. 低置信度案例需要人工复核
